# Logs

A lightweight memory for the project: decisions we took, meeting notes, and issues/risks to watch. Keep entries short and dated so anyone can catch up fast.

!!! tip "How to log effectively"
- One decision per bullet with a date and reason
- Link to related pages (feature, standard, or system)
- Capture risks early; add an owner and a next check date
