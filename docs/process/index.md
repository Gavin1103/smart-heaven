# Process

How we plan, build, review, and improve every sprint. We keep it lightweight but disciplined so each demo works reliably and the city stays cohesive.

!!! info "Sprint cadence"
- Onboarding sprint (3 weeks), then 4 sprints × 3 weeks each
- One team, rotating roles each sprint (no fixed sub‑teams)
- Demo at the end of every sprint; aim for 10× consecutive runs without fixes
